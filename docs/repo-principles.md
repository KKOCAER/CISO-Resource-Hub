# Repository Principles

## 1. Decision-Oriented
This repo should help leaders and practitioners make better decisions, not simply consume more content.

## 2. Architecture First
Security controls and products should be mapped to an operating model and target-state architecture.

## 3. Operationally Grounded
A good document should still make sense at 02:00 during a live incident.

## 4. Vendor-Agnostic by Default
Vendor examples are welcome when they clarify implementation. The design should not depend on brand names.

## 5. Industry Aware
Banking and telco each have distinct exposure, terminology, and regulatory expectations.

## 6. Public but Useful
Because this is a public repository, write for broad reuse while avoiding environment-specific secrets or fragile detail.
