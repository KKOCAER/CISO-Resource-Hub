# Privileged Access Model

## Objectives
- separate admin identity from user identity
- reduce standing privilege
- raise assurance for sensitive operations
- improve accountability

## Core Controls
- separate admin accounts
- phishing-resistant MFA
- PAM
- just-in-time elevation
- hardened admin workstations
- session logging where possible

## Common Failure
Treating privileged access as a role rather than a high-risk operating mode.
