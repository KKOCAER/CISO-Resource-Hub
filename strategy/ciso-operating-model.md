# CISO Operating Model

A CISO function should be designed around outcomes, not org charts.

## Core Functions
- governance and risk
- security architecture
- security engineering
- security operations
- incident response
- resilience
- awareness
- third-party security

## Leadership Outcomes
A mature CISO organization should be able to:
- explain risk in business language
- prioritize control investments
- respond under pressure
- improve security without slowing the company unnecessarily

## Simple Operating Model
### Control Plane
- policy
- risk
- architecture
- standards
- executive communication

### Execution Plane
- engineering
- SOC
- IR
- cloud security
- appsec
- identity
- vulnerability management

### Assurance Plane
- metrics
- audit response
- control testing
- exercise program
- lessons learned

## Common Failure Modes
- overinvesting in tools without design
- underinvesting in identity and recovery
- reporting activity instead of risk
- treating incidents as purely technical
